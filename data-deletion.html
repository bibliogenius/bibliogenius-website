Suppression des données — BiblioGenius

Dernière mise à jour : 27 mai 2026

BiblioGenius est une application de gestion de bibliothèque personnelle développée par Federico CALO. Cette page explique comment demander la suppression des données associées à votre installation et à votre profil sur le serveur d'annuaire optionnel hub.bibliogenius.org.

1. Données stockées localement sur votre appareil

Toutes les données de votre bibliothèque (livres, prêts, contacts, préférences, identité cryptographique) sont stockées localement sur votre appareil, dans des espaces protégés (SQLite, Keychain/Keystore).

Procédure : désinstaller l'application BiblioGenius depuis votre appareil supprime immédiatement et définitivement toutes ces données locales. Aucune action de notre part n'est requise.

2. Données stockées sur le serveur d'annuaire (hub)

Si vous avez utilisé une fonctionnalité réseau (annuaire public, suivi d'autres bibliothèques, envoi/réception de demandes de prêt), un profil a été créé sur notre serveur d'annuaire. Ce profil contient :

  • un identifiant pseudonyme cryptographique (clé publique, ne révèle pas votre identité) ;
  • le nom que vous avez donné à votre bibliothèque ;
  • optionnellement votre pays et votre ville (si vous avez choisi de les partager) ;
  • la liste des ISBN/titres de votre catalogue (uniquement si votre bibliothèque est listée publiquement) ;
  • des messages chiffrés de bout-en-bout en transit dans votre boîte de réception (le serveur ne peut pas les lire) ;
  • un identifiant matériel haché (modèle de l'appareil, empreinte hashée de l'identifiant Android) utilisé uniquement pour éviter les profils en double après réinstallation.

3. Comment demander la suppression de votre profil hub

Envoyez un email à contact@bibliogenius.org avec pour objet « Suppression de mes données BiblioGenius ».

Pour nous permettre d'identifier votre profil, indiquez dans le message :

  • le nom de votre bibliothèque tel qu'affiché dans l'application, ou
  • l'identifiant public de votre profil (8 premiers caractères du node ID, visible dans l'écran « Profil » de l'application si la fonction réseau est activée).

Délai de traitement : votre demande sera traitée dans un délai maximum de 30 jours à compter de la réception du message. Vous recevrez une confirmation par email une fois la suppression effective.

4. Données supprimées vs conservées

Supprimées immédiatement

  • Profil hub (ligne library_profiles) — nom, identifiants, pays, ville, etc. ;
  • Catalogue ISBN éventuellement publié ;
  • Boîte de réception et messages chiffrés associés ;
  • Relations de suivi (followers / suivis) ;
  • Demandes de prêt en cours impliquant votre profil.

Conservées temporairement après suppression

  • Journaux d'événements anonymisés (hub_events) liés à la maintenance et à la sécurité — supprimés automatiquement après 30 jours.
  • Journaux de tentatives d'inscription (registration_failures) utilisés pour la détection d'abus — supprimés automatiquement après 90 jours.
  • Aucune de ces données ne permet de remonter à votre identité.

Données qui n'ont jamais quitté votre appareil

Les sauvegardes locales de votre bibliothèque, votre clé privée cryptographique, vos prêts personnels, vos contacts et vos préférences ne sont stockés que sur votre appareil. Pour les effacer, désinstallez l'application (voir section 1).

5. Suppression automatique en cas d'inactivité

Les profils dormants depuis plus de 7 jours, sans aucun livre publié, sans boîte de réception et sans relation de suivi, sont automatiquement éligibles à la purge périodique par notre équipe. Cette purge ne s'applique pas aux profils actifs.
